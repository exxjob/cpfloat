static inline int cpfloat_validate_optstruct(const optstruct *fpopts)

Validate fields of optstruct struct for double storage format.

This function checks whether the parameters stored in fpopts are valid when double

is used as storage format.

Possible return values are:

  • -4 The rounding mode specified in fpopts->round does not correspond to a valid choice, thus no rounding will be performed.

  • -2 The required number of digits in fpopts->precision is between 26 and 53 inclusive, which might cause double rounding if round-to- nearest is used.

  • -1 The string in fpopts->format is not valid. This is not an error as this value is not used by the C functions, but only by the MEX interface.

  • 0 All the parameters in fpopts are valid.

  • 2 The required number of digits in fpopts->precision is larger than 53 , the number of significant digits in a variable of type double.

  • 3 The required maximum exponent in fpopts->emax is larger than 1023 , the largest possible exponent for a variable of type double.

  • 5 The value of fpopts->flip indicates that soft errors should be introduced, but fpopts->p is not a real number between 0 and 1 and thus does not represent a valid probability.

Errors take precedence over warnings, thus a nonpositive return value implies no errors. In case of multiple issues, the return value is that of the first error (or warning, if no error is present) encountered in the order given in the list above.

Parameters:

fpopts – [in] Parameters describing the target format, the rounding mode, and the probability of soft errors striking the rounded values.

Returns:

The function returns 0 if all the parameters are valid, and a positive number if at least one of them is not. A negative number should be understood as a warning, and indicates that a CPFloat function will return 0 if fpopts is used as fourth argument, but might not perform as intended.
